# 1.定义

C语言编写的一个分布式版本控制系统. 类似的有SVN.
管理多次修改版本修改记录和内容.
集中式版本控制(svn和csv): 先向中心下载,然后本地修改,修改完成之后推送给中心;
分布式版本控制: 一个充当中心的服务(实际没有中心概念),本地就是仓库,每次修改完推送给其他人的本地,可以看到对方的修改内容

# 3.GIT使用

版本库又名仓库，英文名repository

# 3.1.GIT初始化（创建版本库）

# open [git bash]
# 显示当前路径，一般默认是C盘
$ pwd
/c/Users/邵敏杰

# 切换到其他盘，例如D盘
$ cd -L d:

# 查看当前文件夹下文件ll命令和ls命令，ll清晰点
$ ll

# 进入需要进入的文件夹，强烈建议不带中文;cd [file]
$ cd index_shao

# 新建目录 mkdir [file]
$ mkdir githome

# 进入新建的版本库目录
$ cd githome/

# 初始化GIT
$ git init

# 初始化成功会有个.git文件，一般隐藏，可用ls -ah或者ll -ah
$ ll -ah
total 4.0K
drwxr-xr-x 1 邵敏杰 197610 0  8月 20 00:18 ./
drwxr-xr-x 1 邵敏杰 197610 0  8月 20 00:18 ../
drwxr-xr-x 1 邵敏杰 197610 0  8月 20 00:18 .git/

# 3.2.try first GIT demo

在GIT版本库下新建README.txt

# 创建一个README.RD
$ vi README.RD
# 弹出内容,输入文字,i输入文字,ESC退出输入,ZZ保存并退出编辑,具体可以学习vim/v相关知识

# 使用命令git add <file>，注意，可反复多次使用，添加多个文件；
$ git add README.rd
warning: LF will be replaced by CRLF in README.rd.
The file will have its original line endings in your working directory

#使用命令git commit -m <message>,提交所有的add。
$ git commit -m "first demo for git"
[master (root-commit) 1a55f22] first demo for git
 1 file changed, 7 insertions(+)
 create mode 100644 README.rd

# 3.3.版本回退

版本提交错误,进行版本回滚

# 创建个会删除的文件
$ vi needdelfile
$ git add needdelfile
warning: LF will be replaced by CRLF in needdelfile.
The file will have its original line endings in your working directory

$ git commit -m "add file need del"
[master aa338d4] add file need del
 1 file changed, 1 insertion(+)
 create mode 100644 needdelfile

# 4.2.问题及方案

# 问题1）无法上传或更新

问题：即使配置了user.name和user.email推送到github还是提示错误如下

Please tell me who you are.
Run
git config --global user.email "you@example.com"
git config --global user.name "Your Name"
to set your account's default identity.
Omit --global to set the identity only in this repository.
fatal: unable to auto-detect email address (got 'index@Index_Shao.(none)')

解决方案：

# 最简单的是配置github的用户名（git右上角图像下显示的`Signed in as XXX`）
# email就是你的github登录邮箱
git config --global user.email "you@example.com"
git config --global user.name "Your Name"
# 查看配置结果
git config --list

上述解决方案的同等替换方式，找到项目工程目录的.git文件夹，打开之后找到config文件，在最后边加上一句话

[user] email=your email name=your name

# 上面的方法仍然失败了的话，配置gitbash和github的通信协议 ，输入ssh-keygen –t rsa –C “邮箱地址” 然后一直按回车回车回车回车
ssh-keygen -t rsa -C "index_shao@163.com"

执行成功之后提示内容如下,找到标红路径下的文件id_rsa.pub，打开之后，将本机的ssh秘钥添加到个人账户中，打开github自己的主页Settings->SSH and GPG keys-> SSH keys->newSSHkey,将内容复制到KEY中。
